## 1. AI 시대에 필요한 능력: AI 리터러시

AI 리터러시는 AI 도구를 많이 써보는 능력만을 뜻하지 않습니다.  
업무 목적에 맞게 문제를 정의하고, 필요한 맥락을 설계하고, 결과물을 판단하는 능력에 가깝습니다.

발표자료에서는 AI를 다루는 능력을 다음 5가지로 정리했습니다.

| 능력 | 의미 |
|---|---|
| 문제 형상화 | 모호한 목표를 AI가 실행 가능한 정교한 세부 작업으로 설계하는 능력 |
| 맥락 설계 | AI가 최적의 결과를 낼 수 있도록 배경지식과 제약 조건을 설계하는 능력 |
| 변화 적응력 | 새로운 기술과 업무 패턴을 빠르게 수용하고 역할을 재설계하는 능력 |
| 에이전트 활용 | 여러 AI 에이전트 간의 협업 방식을 결정하고 조율하는 능력 |
| 심미적·윤리적 판단 | 결과물의 품질을 시각적·도덕적 관점에서 검증하고 최종 승인하는 능력 |

전사 AX 가속화는 거창한 변화에서만 시작되지 않습니다.  
일상 업무에서 i-Nova를 제대로 이해하고 활용하는 것부터 시작할 수 있습니다.

이 글에서 특히 집중할 부분은 **문제 형상화**와 **맥락 설계**입니다.  
어떤 질문을 어떤 단위로 나눌지, 어떤 자료를 어떤 목적으로 참고시킬지, 대화가 길어질 때 언제 새 세션으로 넘길지 모두 AI 리터러시의 일부입니다.

## 3. 이런 경험, 한 번쯤 있으시죠?

i-Nova를 쓰다 보면 다음과 같은 상황을 만날 수 있습니다.

| 익숙한 장면 | 관련된 문제 |
|---|---|
| "파일을 첨부했는데 자꾸 엉뚱한 답변을 해요." | 파일 첨부와 참고자료 선택 문제 |
| "대화가 길어질수록 앞 내용을 잊어버리는 것 같아요." | 대화 지속성과 컨텍스트 누적 문제 |
| "답변이 중단됐어요." | 한 번에 처리해야 할 맥락이 커진 문제 |
| "답변을 불러오지 못했어요." | 컨텍스트 창 초과나 응답 처리 실패 가능성 |

이 장면들은 모두 i-Nova의 작동 원리를 알면 줄일 수 있는 문제입니다.

핵심은 간단합니다.  
i-Nova는 모든 내용을 무한히 기억하는 도구가 아니라, 제한된 맥락 안에서 참고자료를 조합해 답변하는 도구입니다.

그래서 좋은 결과는 질문 한 문장에서만 나오지 않습니다.  
자료를 어떻게 나누고, 맥락을 어떻게 정리하고, 언제 세션을 전환하느냐에서 답변 품질이 달라집니다.

## 4. i-Nova를 이해하기 위한 3가지 원리

### 원리 1. 입력 데이터는 학습이 아니라 참고입니다

i-Nova는 이미 학습이 끝난 모델입니다.  
질문을 입력하거나 파일을 첨부한다고 해서 그 내용이 모델에 새로 학습되는 것은 아닙니다.

입력 데이터는 모델을 다시 훈련시키는 재료가 아니라, **이번 답변을 만들기 위한 참고 자료**입니다.

그래서 "파일을 넣었으니 알아서 다 반영하겠지"라고 기대하기보다, 다음처럼 목적을 함께 알려주는 것이 좋습니다.

```markdown
첨부한 파일 중 3페이지의 캠페인 목표와 7~9페이지의 성과 데이터를 중심으로,
경영진 보고용 제안서 초안을 작성해줘.
```

파일을 넣는 것보다 중요한 것은, 그 파일을 **어떤 목적으로 읽어야 하는지** 알려주는 것입니다.

### 원리 2. 모델에는 기억할 범위가 있습니다

컨텍스트 창은 대규모 언어 모델이 한 번에 고려하거나 기억할 수 있는 토큰 단위의 텍스트 범위입니다.

쉽게 말하면, i-Nova가 한 번에 펼쳐놓고 볼 수 있는 작업대입니다.  
작업대가 무한하지 않기 때문에 대화가 길어지고 파일이 많아지면 중요한 지시나 근거가 뒤로 밀릴 수 있습니다.

컨텍스트 창의 한계를 넘으면 다음과 같은 현상이 생길 수 있습니다.

- 답변이 중단됨
- 답변을 불러오지 못함
- 앞에서 정한 기준을 잊은 것처럼 보임
- 파일을 첨부했는데 필요한 내용이 충분히 반영되지 않음

이럴 때는 같은 채팅에서 계속 밀어붙이는 것보다, 핵심 맥락을 요약해 새 채팅으로 넘기는 편이 더 안정적입니다.

### 원리 3. AI는 파일 전체를 그대로 읽지 않습니다

파일을 첨부하면 AI가 파일 전체를 처음부터 끝까지 그대로 읽고 답변한다고 생각하기 쉽습니다.  
하지만 실제로는 파일을 작은 조각으로 나누고, 질문과 관련 높은 핵심 조각을 선별해 답변에 참고하는 방식에 가깝습니다.

따라서 "파일 안에 있다"와 "이번 답변에 반영됐다"는 다른 문제입니다.

질문이 모호하면 AI가 엉뚱한 조각을 참고할 수 있습니다.  
반대로 파일 범위, 목적, 출력 형식을 명확히 지정하면 필요한 조각이 답변에 반영될 가능성이 높아집니다.

## 5. 실전 Tip 5가지

### Tip 1. 의도가 구체적일수록 결과는 정교해집니다

좋은 프롬프트는 예쁜 문장이 아니라 업무 인수인계서에 가깝습니다.  
목적, 역할, 참고자료, 강조사항을 분명히 줄수록 결과가 안정됩니다.

**AS-IS**

```markdown
내용 읽고 제안서 초안 좀 잡아줘.
```

이 요청은 간단하지만 의도와 맥락이 부족합니다.  
어떤 제안서인지, 누구 관점인지, 무엇을 강조해야 하는지 알기 어렵습니다.

**TO-BE**

```markdown
신규 무선청소기 캠페인을 위한 내부 미팅 메모와 제품 사양 정보를 바탕으로,
데이터 기반 미디어 전략가 관점에서 매체 집행 전략 및 소재 기획안을 작성해줘.

반드시 아래 내용을 반영해줘.
1. 작년 캠페인의 낮은 CTR 문제를 해결할 것
2. 경쟁사 대비 실생활 밀착형 소구점을 강조할 것
3. 경영진이 빠르게 판단할 수 있도록 결론부터 제시할 것
4. 결과물은 아티팩트 형식의 제안서 초안으로 작성할 것
```

같은 요청이라도 이렇게 바꾸면 결과가 달라집니다.

| 기존 방식 | 개선된 방식 |
|---|---|
| 정보 나열식 답변 | 경영진 맞춤형 결론 중심 구성 |
| 평면적인 구조 | 과거 분석 기반의 매체별 전략 |
| "CTR 향상" 같은 단순 목표 | 소비자 인사이트와 비주얼 제안까지 포함 |

### Tip 2. 한 번에 다 주지 말고, 단계를 나눠 전달하세요

자료가 많을 때는 한 번에 전부 넣기보다, 먼저 로드맵을 선언하는 것이 좋습니다.

**STEP 01. 로드맵 선언**

```markdown
너는 전략 기획자야.
총 3번에 걸쳐 참고자료를 보낼 거야.
1~2회차에는 데이터를 인지만 하고, 3회차까지 받은 뒤 최종 회의록을 작성해줘.
```

**STEP 02. 순차 입력**

```markdown
이게 1번째 데이터인 [A]야.
주요 안건 3가지만 도출해서 기억해두고 다음 데이터를 기다려줘.
```

**STEP 03. 최종 결과 도출**

```markdown
데이터가 모두 전달됐어.
입력된 내용을 바탕으로 경영진 보고용으로 논리적이고 간결한 회의록 아티팩트를 생성해줘.
```

이 방식은 다음과 같은 자료를 다룰 때 유용합니다.

| 자료 유형 | 나누는 단위 예시 |
|---|---|
| 회의록 | 오전/오후, 안건별 |
| 리서치 | 트렌드, 타겟, 경쟁사 |
| 매체 제안 | 네이버, 카카오, 메타 |
| 사내 규정 | 인사, 복지, 보안 |
| 프로젝트 문서 | 기획, 개발, 테스트 |

### Tip 3. 대화가 길어지면 새 채팅으로 리셋하고, 요약으로 핸드오프하세요

#### Part 1. 주제별 세션 전환

대화가 길어질수록 컨텍스트 창에 쌓이는 정보도 많아집니다.  
이때 답변이 느려지거나, 중단되거나, 앞의 기준을 놓치는 것처럼 보일 수 있습니다.

새 채팅 전환을 권장하는 경우는 다음과 같습니다.

- 오전 업무와 오후 업무의 주제가 달라졌을 때
- 회의록 작성에서 제안서 작성으로 목적이 바뀌었을 때
- 답변이 느려지거나 전송 실패, 응답 지연이 반복될 때
- 이전 파일과 현재 질문이 섞여 답변이 흔들릴 때

새 채팅은 단순히 대화를 끊는 것이 아닙니다.  
컨텍스트를 초기화해서 현재 작업에 필요한 맥락만 다시 올리는 운영 방식입니다.

#### Part 2. 세션 간 핸드오프

새 채팅을 열면 이전 대화의 맥락은 자동으로 이어지지 않습니다.  
대신 마무리 단계에서 핸드오프용 요약을 만들면 작업의 연속성을 유지할 수 있습니다.

마무리할 때는 이렇게 요청합니다.

```markdown
지금까지 우리가 나눈 대화의 핵심 목적과 주요 설정값을 요약해줘.
다음 세션에서 이 맥락을 그대로 이어갈 수 있도록 정리해줘.
```

그 다음 새 채팅 첫 메시지에 이렇게 붙여넣습니다.

```markdown
앞선 세션의 요약본이야.
이 맥락을 바탕으로 이어서 대화하자.
```

이 방식은 이전 세션의 핵심 로직과 목적을 유지하면서도, 새 세션의 컨텍스트 창을 더 깔끔하게 확보할 수 있게 해줍니다.

### Tip 4. 폴더 기능으로 팀 공용 프롬프트를 자산화하세요

잘 만든 프롬프트를 개인 대화 목록에만 남겨두면 다시 찾기 어렵습니다.  
반복해서 쓰는 업무라면 폴더를 활용해 팀 공용 자산으로 관리하는 편이 좋습니다.

| 기존 방식 | 개선된 방식 |
|---|---|
| 개인 대화 목록에 흩어져 있음 | 검증된 고성능 프롬프트를 조직 자산으로 공유 |
| 카테고리 분류용으로만 사용 | 실무에 바로 투입 가능한 레퍼런스로 재사용 |
| 폐쇄적이고 휘발적임 | 공유되고 축적되는 업무 자산이 됨 |

폴더를 사용할 때 중요한 점도 있습니다.  
**폴더 안에서 새 대화를 시작하면, 개인 설정 프롬프트보다 폴더 프롬프트가 우선 적용됩니다.**

따라서 팀 공통 기준, 보고서 톤, 산출물 형식을 맞추는 용도로 활용하기 좋습니다.

### Tip 5. PROMPT 6요소로 마지막 점검하세요

프롬프트를 작성할 때는 PROMPT 6요소를 점검하면 좋습니다.

| 요소 | 의미 | 예시 |
|---|---|---|
| P | Persona, 역할 | 10년 차 마케터 |
| R | Reference, 참고자료 | 타겟 분석 리포트 |
| O | Objective, 실행 목적 | 신규 광고주 수주 |
| M | Mode, 출력 형식 | 매체 제안 아티팩트 |
| P | Point of View, 대상 관점 | 광고주 팀장 눈높이 |
| T | Tone, 어조 | 데이터 중심 설득조 |

**AS-IS**

```markdown
인스타 광고 소재 아이디어 좀 짜줘.
```

**TO-BE**

```markdown
당신은 10년 차 마케터입니다.
첨부한 타겟 리포트를 참고하여, 신규 광고주 수주를 목적으로,
광고주 팀장 관점에서 이해하기 쉬운 매체 제안 아티팩트를 작성해줘.
톤은 데이터 중심의 설득조로 유지해줘.
```

공식 자체를 외우는 것보다 중요한 것은, AI가 일을 수행하는 데 필요한 맥락을 빠뜨리지 않는 것입니다.

## 6. 오늘의 정리

i-Nova는 원리대로 쓰면 결과가 달라집니다.

| 구분 | 정리 |
|---|---|
| 원리 이해 | 입력 데이터는 학습이 아니라 참고용입니다. 대화나 파일이 길어지면 컨텍스트 창 초과로 오류가 발생할 수 있습니다. |
| 습관 변화 | 주제가 바뀌면 새 채팅으로 세션을 전환하고, PROMPT 6요소로 명확하게 지시합니다. |
| 결과 향상 | 좋은 프롬프트와 산출물은 폴더와 아티팩트, 개인 백업을 통해 조직의 지식 자산으로 축적할 수 있습니다. |

답변이 아쉬울 때는 같은 질문을 반복하기 전에 먼저 확인해보세요.

- 목적이 분명한가?
- 참고해야 할 파일 범위를 지정했는가?
- 대화가 너무 길어지지는 않았는가?
- 새 세션으로 넘길 요약이 준비되어 있는가?
- 잘 나온 프롬프트를 팀 자산으로 남겼는가?

i-Nova의 답변 품질은 모델 성능만이 아니라, 사용자가 어떤 맥락을 어떻게 설계하느냐에 크게 좌우됩니다.  
원리를 이해하고 쓰면, i-Nova는 더 안정적인 업무 파트너가 될 수 있습니다.

## 추가 안내: i-Nova 실험실 1.0.0 설치

금요스터디의 본 내용은 i-Nova를 제대로 이해하고 사용하는 방법입니다.  
그 다음으로, 이번 자리에서는 i-Nova 실험실 1.0.0도 함께 안내합니다.

i-Nova 실험실은 i-Nova 대화 화면 오른쪽에 실험실 패널을 더하는 내부 Chrome 확장입니다.  
1.0.0 버전은 일반 사용자가 i-Nova를 더 편하게 점검하고 활용할 수 있도록 제공되는 전사 배포 버전입니다.

사용자가 바로 체감할 수 있는 주요 기능은 다음과 같습니다.

| 기능 | 활용 방향 |
|---|---|
| 대화 탐색 | 현재 대화의 질문 흐름을 모으고, 대화 안에서 필요한 질문을 빠르게 찾습니다. |
| 예상 컨텍스트 | 지금 대화가 얼마나 길어졌는지 참고 신호로 확인하고, 새 채팅 전환 시점을 판단하는 데 활용합니다. |
| 프롬프트 검토 | 입력한 프롬프트의 부족한 점을 확인하고, 다듬은 프롬프트를 복사하거나 입력창에 반영합니다. |
| 프롬프트 보관함·스토어 | 자주 쓰는 요청을 저장하고, 검증된 프롬프트를 찾아 업무에 재사용합니다. |
| 회의 룸 | 회의 작업실로 이동해 회의 기록과 후속 정리를 이어갑니다. |
| 릴리스 안내 | 현재 버전, 최신 배포 안내, 다운로드 흐름을 확인합니다. |

앞에서 정리한 i-Nova 사용 원리를 먼저 이해한 뒤 실험실 1.0.0을 함께 활용하면, 긴 대화를 점검하고, 프롬프트를 다듬고, 좋은 요청을 재사용하는 흐름을 더 일관되게 이어갈 수 있습니다.

### 설치 방법

아래 순서로 설치하면 됩니다.

1. [아이노바 실험실 최신 배포 파일](https://browser-extension-v2.web.app/extension-v2/downloads/latest.zip)을 다운로드합니다.
2. 다운로드한 ZIP 파일의 압축을 해제합니다.
3. Chrome 주소창에 `chrome://extensions`를 입력해 확장 프로그램 관리 화면으로 이동합니다.
4. 오른쪽 상단의 **개발자 모드**를 켭니다.
5. **압축해제된 확장 프로그램을 로드**를 클릭합니다.
6. 압축을 해제한 폴더 중 `manifest.json` 파일이 보이는 폴더를 선택합니다.
7. `https://inova.incross.com/`에 접속한 뒤, 화면 오른쪽에 i-Nova 실험실 패널이 표시되는지 확인합니다.

이미 이전 버전을 설치한 경우에는 최신 배포 파일을 다시 다운로드해 압축을 해제한 뒤, `chrome://extensions`에서 기존 i-Nova 실험실을 제거하고 새 폴더를 다시 로드하면 됩니다.
